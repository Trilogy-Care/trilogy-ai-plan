Who is Elias

A personal assistant, not a product

Elias is a personal AI assistant built by and for Imran Mehmood, AI Innovation Lead in Trilogy Care's Growth function. It was named on 12 May 2026 — the "quiet counsellor" archetype: low-key, steady, does the work, doesn't perform.

Elias runs inside Claude Code on Claude Opus 4.7 (1M-token context). It is not a product and not a team tool — it is one operator's assistant, designed end to end for a single person's workflow. That narrow scope is deliberate: everything Elias does is shaped to how Imran actually works.

Four things that make it real

The honest pillars
01
A memory that persists
Elias does not start each session blank. A set of ~50 markdown memory files load every session — facts about the operator, behavioural rules, project state, and references. The memory is the continuity; the model is just the engine.
02
An approval gate
Elias never acts on anything consequential without Imran's explicit tap. Work surfaces as ✅ / ❌ / 🕒 approval cards on Telegram. Approve and it proceeds; deny and it is recorded; defer and it waits. Elias drafts and proposes — Imran decides.
03
Authority to do real work
Within that gate, Elias does the work, not just talk about it: it writes code, opens pull requests, restructures sites, runs autonomous overnight agents across multiple repositories.
04
It improves itself
An auto-rule-extractor turns Imran's corrections into durable behaviour. The whole memory + config state is snapshotted to a private git repo. Elias is measurably more useful than it was two weeks ago — not because the model changed, but because the memory got denser.

The memory system

One fact per file

Elias's memory is a set of small markdown files, one fact per file, in four types:

user
Who Imran is, how he works, his preferences.
feedback
Behavioural rules captured from his corrections.
project
Live workstream state across GROW, TCPW and more.
reference
Pointers to systems, repos, credentials locations.

A single MEMORY.md index loads at the start of every session. The intelligence is not weight updates — it is disciplined file-writing: one fact per note, human-readable, version-controlled, snapshotted weekly. Swap the model and the memory still works.

The Telegram bridge

The surface Imran already has

Elias reaches Imran on Telegram — the surface he already has on his phone.

💬
Two-way bot
Imran messages Elias; Elias replies and acts.
✅
Approval cards
Every consequential action posts a card with ✅ Approve / ❌ Deny / 🕒 Later buttons. Tapping resolves it.
🎤
Voice notes
Imran sends a voice message; Elias transcribes it and acts on the intent.
☼
Morning digest
A daily brief of what is pending, what shipped, what needs a decision.

Six background jobs keep it running: sweep, worker, telegram, digest, liveness, and a CI watcher — all launchd jobs on Imran's machine.

The Fireflies sweep

Meetings into tracked work

Elias watches Imran's meeting transcripts via Fireflies. After a meeting it sweeps the transcript for commitments and action items, files them into an inbox, and drafts the follow-up work. Nothing is sent — drafts wait for review. This turns "things said in a meeting" into "tracked, drafted, ready to approve".

The Elias Queue

Built 22 May 2026

Approving a Telegram card no longer just records a decision — it creates a tracked GitHub issue in a backlog repo (Trilogy-Care/elias-queue) and adds it to a project board with a Triage → Building → In PR → Done flow.

Denied cards become closed, labelled issues — a searchable record of decisions not to do something. The triage happens on a phone; the work lands as a structured backlog.

What Elias is NOT

Saying only what is true

In the spirit of saying only what is true:

✗ Elias is not an AGI and makes no claim to be.
✗ There is no 1,000-agent forest — Elias spawns a small number of task-scoped sub-agents when a job genuinely parallelises, and that is all.
✗ There is no separate "ASI" tier, no meta-intelligence above it.
✗ There is no production benchmark suite, no published eval scores.
✗ Elias is not a multi-user platform — it serves exactly one operator.

Elias is a focused, honest, single-operator assistant. Everything above is something you could open the repo and verify. That is the design.
